## 2026-07-13：企业产品册公开联系方式

### 需求

- 游客产品册企业标题卡新增“联系方式”；点击后才显示企业在工作台维护的电话号码，未填写时不显示入口。
- 企业公开电话必须与平台用户管理中的联系人、备注完全区分，不能复用或公开平台内部资料。

### 实现

- `companies` 新增 `public_contact_phone`，空字符串表示不公开；`contact_name/contact_note` 继续作为平台内部联系人和内部备注。Schema 同时包含新建结构、幂等加列、50 字符命名约束和字段注释。
- 新增 `PATCH /api/admin/company-contact`，只接受企业 ID 与公开电话；先校验后台 session，再校验企业范围，允许当前企业或平台管理员代管，支持清空，并拒绝超长、控制字符和少于 5 位数字的非空号码。
- 公开公司最小查询只增加 `public_contact_phone`，不选择登录账号、内部联系人或内部备注；保存后立即失效公开 company/catalog 缓存。
- 企业产品工作台顶部新增“公开联系方式”入口，展开“产品册公开电话”面板；帮助文案明确与平台内部资料无关。平台用户管理表单同步改标“内部联系人/内部备注”，公开电话不在该表单维护。
- 游客产品册仅在公开电话非空时渲染“联系方式”按钮；号码默认隐藏，点击后显示原格式电话与可拨打 `tel:` 链接，再次点击收起。弹层覆盖显示，不挤压默认产品网格。
- 同步更新 `src/types.ts`、演示数据、公开/后台查询、README、AGENTS 与开发框架。

### 验证

- `git diff --check`、TypeScript `--noEmit` 与 Next.js 16.2.10 生产构建通过，新路由 `/api/admin/company-contact` 正常进入构建产物。
- 隔离的无 Supabase 本地运行态中，未登录更新公开电话返回 401；企业登录后因无 Supabase 返回 503，不会用演示数据假成功。
- 390x844 与 508x746 浏览器回归通过：公开电话按钮可见、默认收起，点击后显示 `138 0000 0000`，拨号地址为 `tel:13800000000`，再次点击可收起；两个视口默认首屏均仍有 6 个产品卡。
- 公开页面 DOM 不含演示内部联系人和内部备注；平台用户管理明确显示“内部联系人/内部备注”，企业账号和平台管理员进入企业工作台时均能看到独立公开电话面板；控制台无运行错误。

### 部署注意

- 这是增量数据迁移。生产部署前必须先在当前 Supabase 项目执行更新后的 `supabase/schema.sql`，新增 `public_contact_phone` 后再发布代码；未迁移数据库时，新版公开公司查询会按既有 fail-closed 规则拒绝展示。

## 2026-07-13：统一后台登录入口

### 需求

- 访问主域名根路径时直接显示账号和密码，不再先展示产品册说明卡。
- 不同企业使用各自账号密码后，应自动进入各自企业后台；平台管理员仍能进入平台后台。
- 保留原平台专用登录和企业专属管理链接，不暴露企业编号、名称或访问 slug。

### 实现

- 根路径 `/` 改为统一登录入口，账号默认留空；已有平台或企业 session 时直接返回对应后台。
- `LoginForm` 增加统一、平台、企业三种明确模式；统一模式读取登录 API 返回的站内目标并使用替换式导航，且拒绝空目标、协议相对地址和非站内地址。
- `/api/admin/login` 的统一模式先校验完整平台凭据，再按规范化且唯一的 `companies.login_username` 精确定位企业并校验 PBKDF2 密码；成功后只由服务端返回 `/admin` 或数据库中不透明 slug 对应的后台地址。
- 账号不存在、密码错误和企业不匹配继续使用统一错误；原 `/admin/login` 保持平台专用，原 `/:companySlug` 登录保持企业范围校验与 session 租户隔离。
- 登录页增加 44px 输入和按钮高度、`100dvh` 与纵向滚动支持，在 508x746、390x844 和软键盘场景下保持表单可见。
- 同步更新 `README.md`、`AGENTS.md` 和 `DEVELOPMENT_FRAMEWORK.md`，固化统一入口的路由、数据流、安全边界和回归要求。

### 验证

- TypeScript `--noEmit`、`git diff --check` 和 Next.js 16.2.10 生产构建通过；根路径保持动态渲染以读取 HTTP-only session。
- 隔离的无 Supabase 本地运行态中，统一入口使用 `demo/demo123` 正确进入 `/demo-catalog`，使用 `admin/admin123` 正确进入 `/admin`；错误凭据留在 `/` 并只显示“账号或密码不正确”。
- `/admin/login` 仍显示平台管理员专用表单，`/demo-catalog` 未登录时仍显示企业专属表单；登录 API 的目标地址仅由服务端角色或数据库企业记录生成。
- 508x746 与 390x844 浏览器回归中，账号、密码和登录按钮均在首屏完整可见；390x844 公共产品册仍有 6 个产品卡进入首屏。

## 2026-07-12：企业编号隐私、到期门禁与管理员图片修改

### 需求

- 顺序厂号不再出现在游客或企业后台的 URL、页面、浏览器标题和下载文件名中。
- 企业停用或按北京时间到期后，游客访问目录和详情均不可见，不再展示企业名称或到期状态。
- 平台管理员可从用户管理后台进入任一企业产品工作台，查看并替换产品图片。

### 实现

- `company_number` 继续作为数据库内部排序字段；新企业由 API 和数据库触发器生成 128 位随机 `site-...` slug，Schema 幂等迁移旧 `c001` 形式。根路径改为中性入口，后台文案、metadata、企业列表和报价文件名不再渲染 slug。
- 公开目录和详情改为每请求动态执行门禁，按 `Asia/Shanghai` 判断到期；未知、停用和过期企业统一 404，并设置 `noindex,nofollow`。Supabase 已配置后的公开查询失败不再回退演示数据。
- `/admin` 企业行新增“管理产品/图片”和“公开产品册”；平台管理员进入企业工作台后可返回用户管理。产品编辑器复用新增/上传时的 WebP 压缩和 R2 PUT 流程，支持当前图、新图预览和上传进度。
- 产品 PATCH 要求图片 URL、对象键、宽高成组提交，严格校验 R2 公开地址、当前企业 object key 前缀和尺寸；保留会话鉴权、企业隔离及公开缓存失效。
- 同步更新 `README.md`、`AGENTS.md` 和 `DEVELOPMENT_FRAMEWORK.md`，将随机访问标识、到期 fail-closed 和平台管理员图片权限固化为后续基线。

### 验证

- TypeScript `--noEmit`、`git diff --check` 和 Next.js 16.2.10 生产构建通过；公开目录与详情均为动态路由。
- 北京时间边界验证通过：到期日 23:59:59 仍有效，次日 00:00:00 立即失效；停用企业不可访问。连续生成 1000 个 `site-...` slug，格式正确且无重复。
- 隔离的无 Supabase 演示运行态中，`/`、`/c/demo-catalog`、`/c/demo-catalog/p/TC-001` 返回 200，未知企业返回 404，页面不含 `c001` 或到期提示。
- 浏览器真实点击验证：平台管理员可进入企业图片工作台、返回用户管理，并在选中 `HW-001` 后打开当前图与替换图片控件；390x844 首屏可见 8 个产品卡。

### 部署注意

- 为避免短暂中断，先部署包含随机 slug 支持的代码，再立即在 Supabase SQL Editor 执行更新后的 `supabase/schema.sql`。迁移会使旧编号链接和旧企业会话立即失效，应从 `/admin` 重新打开并分发新链接。

## 2026-07-11：中国大陆访问性能深度优化

### 需求

- 减少公开产品册从中国大陆访问时的跨境数据库请求和动态渲染开销。
- 将 Vercel Function、Supabase 与 R2 图片链路迁往或对齐亚太区域，并为图片启用正式 CDN 域名。
- 后台写入成功后立即刷新公开缓存，同时保持后台、登录和报价单数据不进入公共缓存。

### 实现

- 将公开企业、产品册列表和单产品详情拆成独立最小查询；详情页不再读取完整产品列表。
- 使用 `React.cache()` 做请求内去重，并用 Next.js Data Cache 按企业标签做 300 秒跨请求缓存。
- 两条公开路由启用 `force-static` 和 300 秒 ISR；企业、分类、产品写接口成功后精确失效标签与路径。
- 公开类型和演示回退均显式裁剪，只向客户端传递搜索、筛选、卡片和详情所需字段。
- 新增公开产品列表组合索引、`img.huowu.org` 图片白名单和 Vercel 东京 `hnd1` 区域配置；保留旧 `r2.dev` 域名用于七天回滚。

- 本地生产构建首次访问产品册返回 `x-nextjs-cache: MISS`，第二次返回 `HIT`，并带 `s-maxage=300`。

### 线上迁移
